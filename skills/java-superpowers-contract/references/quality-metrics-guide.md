# 数据质量三指标判定指南

## 指标定义

### 1. NULL 率
字段值为 NULL 的记录占比。
- **> 80%**: 字段在当前业务场景下基本未被使用，考虑是否冗余
- **50% - 80%**: 可选字段，但大量缺失可能意味着业务上不需要该字段
- **< 50%**: 正常范围，视业务而定

### 2. 空字符串率
字段值为 `''` 或 `""` 的记录占比（仅对字符串类型有效）。
- **> 30%**: 设计问题——字段应设 NOT NULL 且提供默认值
- **10% - 30%**: 需检查业务逻辑，是否应该用 NULL 而非空串
- **< 10%**: 正常

### 3. 哨兵值率
字段值等于魔数/哨兵值的记录占比。审计自动识别的哨兵值包括：
- 数值型: `0`, `-1`, `9999`, `999999`
- 日期型: `1970-01-01`, `0000-00-00`, `2099-12-31`
- **> 10%**: 业务层在用魔数表示特殊语义（如"未设置"），建议使用 NULL 或枚举替代

## 综合质量评分

| 评分 | 判定 |
|------|------|
| A (90-100) | 健康 |
| B (70-89) | 良好，有轻微瑕疵 |
| C (50-69) | 需关注 |
| D (0-49) | 需重建 |

评分公式：`100 - (NULL率 * 40 + 空字符串率 * 30 + 哨兵值率 * 30)`，每项超过阈值的部分额外扣分。
